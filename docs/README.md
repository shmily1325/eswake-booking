# 📚 文檔目錄

這個資料夾包含專案的所有技術文檔和指南。

## 📁 資料夾結構

### `/archive/` - 歷史記錄
已完成或已解決的問題記錄，保留作為參考：
- `URGENT_FIX_NEEDED.md` - 緊急修復記錄（已解決）
- `FINAL_CHECK_REPORT.md` - 部署前檢查報告
- `DEPLOYMENT_TEST_RESULTS.md` - 部署測試結果
- `TIMEZONE_FIX_REVIEW.md` - 時區修復回顧
- `CODE_ANALYSIS.md` - 代碼分析報告（2025-11-09）
- `FORMATTER_REPLACEMENT_PLAN.md` - 格式化函數替換計劃（已完成）
- `FORMATTER_REPLACEMENT_PROGRESS.md` - 格式化函數替換進度（已完成）

### `/testing/` - 測試文檔
測試相關的文檔和工具：
- `TEST_CHECKLIST.md` - 完整測試清單
- `TEST_REQUIRES_DRIVER.md` - 駕駛功能測試清單
- `TEST_HELPER.html` - 測試輔助工具

### `/features/` - 功能說明
各功能模組的詳細說明：
- `COACH_REPORT_V2_README.md` - 教練回報系統 V2 說明
- `MEMBERSHIP_FIELDS.md` - 會員欄位說明

### `/` - 架構文檔
系統架構和設計文檔：
- `CoachReport-Architecture.md` - 教練回報架構
- `CoachReport-Logic.md` - 教練回報邏輯
- `CoachReport-Optimization.md` - 教練回報優化
- `CoachOverview-Logic.md` - 教練總覽邏輯
- `CODE_QUALITY_SUMMARY.md` - 代碼品質改善總結

## 🔍 快速導航

## 📝 文檔維護

### 新增文檔時
- 技術架構和設計 → `/docs/`
- 功能說明 → `/docs/features/`
- 測試文檔 → `/docs/testing/`
- 已完成的問題記錄 → `/docs/archive/`
- 用戶指南（設置、使用） → 專案根目錄

### 命名規範
- 使用大寫和底線：`FEATURE_NAME.md`
- 包含日期（如需要）：`REPORT_2025_11_19.md`
- 清楚描述內容：`COACH_REPORT_V2_README.md` ✅ vs `README2.md` ❌
